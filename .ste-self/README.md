# STE Runtime Self-Analysis

This directory contains the complete semantic state graph of ste-runtime analyzing itself.

## Purpose

These are **real, production-quality examples** of AI-DOC slices generated by RECON. They demonstrate:

1. **Self-documenting capability** - ste-runtime can analyze itself
2. **Slice format and structure** - See actual semantic extraction output
3. **E-ADR-009 in action** - Automatic domain discovery (detected "ste-runtime" as a TypeScript project)
4. **Relationship modeling** - Function calls, imports, dependencies

## What's Inside

This self-analysis contains **~866 semantic slices** extracted from ste-runtime's TypeScript source code:

### Graph Slices
- **Modules**: File-level metadata and exports
- **Functions**: Function signatures, parameters, return types
- **Classes**: Class definitions, methods, inheritance

### Behavior Slices
- **Call graphs**: Function invocation relationships
- **SDK usage**: External library and framework usage
- **Environment variables**: Configuration dependencies

## How It Was Generated

```bash
cd ste-runtime
node dist/cli/recon-cli.js --mode=full .
```

RECON automatically:
1. Discovered project structure (Phase 0, E-ADR-009)
2. Found TypeScript source files (Phase 1)
3. Extracted semantic elements (Phase 2-4)
4. Generated and persisted slices (Phase 5)
